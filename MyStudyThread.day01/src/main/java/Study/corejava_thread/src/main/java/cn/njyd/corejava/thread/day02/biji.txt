线程安全问题的解决方法：
第一种采用synchronized同步方式
    1.同步代码块
      语法：synchronized(对象锁）{
         //可能出现线程安全的代码块
      }
      说明：任意的对象都可以作为锁（也称为监视器），只要确保不同的线程拿到的是同一把锁即可，
      同步代码块是需要显示的加锁
    2.同步方法
      语法：修饰符 synchronized 返回值类型   方法名称(参数列表){

          //可能出现线程安全问题的代码块
      }
      说明：同步方法采用的是隐式的锁，对于实例方法采用的是this锁，对于静态方法采用的是当前类 的二进制字节码文件对象作为锁
